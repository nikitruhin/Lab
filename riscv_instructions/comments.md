# Пример 1: 

### Исходный код на C
```c
int main(void) 
{
    register int a = 1;    // записываем в регистровую переменную 1
    return a;               // возвращаем значение из регистровой переменной
}
```

### -O0 (без оптимизации)
```assembly
main:
	# ПРОЛОГ ФУНКЦИИ (подготовка стека)
	addi	sp,sp,-16      # выделяем 16 байт в стеке (стек растет вниз)
	sw	s0,12(sp)        # сохраняем регистр s0 в стек (чтоб потом восстановить)
	sw	s1,8(sp)         # сохраняем регистр s1 в стек (чтоб потом восстановить)
	addi	s0,sp,16       # устанавливаем frame pointer для доступа к переменным

	# КОД: register int a = 1;
	li	s1,1             # загружаем константу 1 в регистр s1

	# ПОДГОТОВКА ВОЗВРАТА: return a;
	mv	a5,s1            # копируем значение из s1 в a5
	mv	a0,a5            # копируем из a5 в a0 (регистр возврата)
	                     # Соглашение о вызовах: результат в a0

	# ЭПИЛОГ ФУНКЦИИ (восстановление стека)
	lw	s0,12(sp)        # восстанавливаем s0 из стека
	lw	s1,8(sp)         # восстанавливаем s1 из стека
	addi	sp,sp,16       # возвращаем указатель стека обратно

	# ВОЗВРАТ
	jr	ra               # прыгаем по адресу в ra (return address)
	                     # Это I-type: jalr x0, ra, 0
```

### Строка: `li s1, 1`
**Означает:** в регистр `s1` было перемещено значение 1.

Это прямое соответствие строке `register int a = 1;`

### Строки: `mv a5, s1` и `mv a0, a5`
**Означают:** сначала значение копируется из `s1` в `a5`, потом из `a5` в `a0`. (связано с **соглашением о вызовах функций** в языке Си, **результат функции возвращается в регистре a0**)


### Строки: `sw s0,12(sp)` и `sw s1,8(sp)`
**Означают:** сохранение регистров `s0` и `s1` в стек.

Cначала компилятор поместил `s0` и `s1` в стек, тем самым запомнил старые значения регистров, а после окончания работы функции (строки `lw s0,12(sp)` и `lw s1,8(sp)`) вернул из стека эти значения обратно в регистры.

### Строки восстановления: `lw s0,12(sp)` и `lw s1,8(sp)`
**Означают:** загрузка обратно сохраненных значений.

### Строка: `jr ra`
**Означает:** возврат из функции по адресу в регистре `ra`.

## Почему нельзя сразу `li a0, 1` => **из уровня оптимизации -O0**.

Мы написали в коде не просто `return 1`, а использовали **регистровую переменную**:
```c
register int a = 1;
return a;
```
Компилятор не должен вырезать код, а мы написали не return 1, мы использовали регистровую переменную. Т. е. компилятор сначала поместил значение в регистр, а затем, следуя соглашению, вернул результат. 

Цепочка: `1 → s1 → a5 → a0`

##  С ОПТИМИЗАЦИЕЙ -O2?

```assembly
main:
	li	a0,1    # сразу загружаем 1 в регистр возврата
	ret
```
Компилятор понял:
- Регистровая переменная не нужна — можно сразу загрузить константу в `a0`
- Лишние перемещения между регистрами можно убрать
- Пролог и эпилог функции не нужны для такой простой функции

Цепочка стала: `1 → a0`

**-O0**: 11 инструкций, видна каждая деталь  
**-O2**: 2 инструкции, только суть

## КЛЮЧЕВАЯ I-TYPE ИНСТРУКЦИЯ

В нашем примере это `li s1, 1`, которая на самом деле является:
```assembly
addi s1, x0, 1
```

Где:
- **addi** — сложение с непосредственным значением (immediate)
- **s1** — регистр назначения
- **x0** — первый операнд (регистр x0 всегда равен 0)
- **1** — непосредственное значение (константа)
Один операнд — **регистр** (x0), а второй — **непосредственное значение** (1), которое встроено прямо в инструкцию.

# Пример 2: 

### Исходный код на C
```c
int main(void) 
{
    int a = 1;      // обычная переменная (будет в стеке)
    int b = a * 2;  // умножение на 2
    return b;       // возвращаем результат
}
```

1. -O0 (БЕЗ ОПТИМИЗАЦИИ)

Ассемблерный код:
```assembly
main:
    addi    sp,sp,-32      # I-type: выделяем 32 байта в стеке
    sw      s0,28(sp)      # S-type: сохраняем s0 (чтобы потом восстановить)
    addi    s0,sp,32       # I-type: устанавливаем frame pointer
    
    li      a5,1           # I-type: загружаем 1 в регистр a5
    sw      a5,-20(s0)     # S-type: сохраняем a5 в стек (это переменная a)
    
    lw      a5,-20(s0)     # I-type: читаем a из стека обратно в a5
    slli    a5,a5,1        # I-type: сдвигаем влево на 1 (умножение на 2)
    sw      a5,-24(s0)     # S-type: сохраняем результат в стек (это переменная b)
    
    lw      a5,-24(s0)     # I-type: читаем b из стека
    mv      a0,a5          # I-type: копируем в a0 (регистр возврата)
    
    lw      s0,28(sp)      # I-type: восстанавливаем s0
    addi    sp,sp,32       # I-type: освобождаем стек
    jr      ra             # I-type: возврат из функции
```
2. С -O2 (С ОПТИМИЗАЦИЕЙ)
Ассемблерный код:
```assembly
main:
    li      a0,2    # I-type: сразу загружаем результат 2
    ret             # I-type: возврат
```
Компилятор вычислил всё на этапе компиляции:

a = 1 (константа)

b = 1 * 2 = 2 (тоже константа)

Интересный момент:  Умножение через сдвиг
a * 2 превратилось в slli a5,a5,1 (т.к. быстрее).

# Пример 3: 

### Исходный код на C
```c
float FastInvSqrt(float x) {
    float xhalf = 0.5f * x;
    int i = *(int*)&x;
    i = 0x5f3759df - (i >> 1);
    x = *(float*)&i;
    x = x * (1.5f - xhalf * x * x);
    return x;
}

int main() {
    float result = FastInvSqrt(3.14159f);
    return (int)(result * 1000);
}
```
Ассемблерный код (RISC-V RV32I)
```assembly
FastInvSqrt:
	lui	a5,%hi(.LC0)        # U-type: загружаем старшие биты 0.5f
	addi	sp,sp,-16           # I-type: выделяем стек
	lw	a1,%lo(.LC0)(a5)     # I-type: загружаем 0.5f из памяти
	sw	s0,8(sp)             # S-type: сохраняем s0
	li	s0,1597464576        # I-type: загружаем 0x5f375000
	srai	a5,a0,1             # I-type: сдвиг вправо (i >> 1)
	addi	s0,s0,-1569        # I-type: добавляем -1569 → 0x5f3759df
	sw	ra,12(sp)             # S-type: сохраняем обратный адрес
	sub	s0,s0,a5             # R-type: ВЫЧИТАНИЕ 
	call	__mulsf3            
	
	mv	a1,s0                # I-type: копируем результат
	call	__mulsf3            
	
	mv	a1,s0                # I-type: копируем
	call	__mulsf3            
	
	lui	a5,%hi(.LC1)        # U-type: загружаем старшие биты 1.5f
	mv	a1,a0                # I-type: копируем
	lw	a0,%lo(.LC1)(a5)     # I-type: загружаем 1.5f
	call	__subsf3            
	
	mv	a1,s0                # I-type: копируем
	call	__mulsf3            
	
	lw	ra,12(sp)             # I-type: восстанавливаем ra
	lw	s0,8(sp)              # I-type: восстанавливаем s0
	addi	sp,sp,16            # I-type: освобождаем стек
	jr	ra                    # I-type: возврат

main:
	li	a0,563               # I-type: результат 0.563 * 1000
	ret                        # I-type: возврат
```
Разбор кода:

1. Загрузка констант (U-type + I-type)
assembly
lui	a5,%hi(.LC0)        # грузим 0x3F000 (0.5f)
li	s0,1597464576       # грузим 0x5F375000
addi	s0,s0,-1569        # добавляем -1569 = 0x5F3759DF
2. Сдвиг (I-type)
assembly
srai	a5,a0,1          # i >> 1 (деление экспоненты на 2)
3. САМАЯ ВАЖНАЯ ИНСТРУКЦИЯ (R-type)
assembly
sub	s0,s0,a5           # 0x5F3759DF - (i >> 1)

4. Вызовы библиотечных функций 
assembly
call	__mulsf3          # умножение для float
call	__subsf3          # вычитание для float
В RV32I нет аппаратного FPU, поэтому вызываются библиотечные функции.
# Пример 4: B-type и J-type инструкции

## Исходный код
```c
int factorial(int n) {
    if (n <= 1)
        return 1;
    else
        return n * factorial(n - 1);
}

int main() {
    return factorial(5);
}
```
Примичание: 0O - дает рекурисвную реализацию 02 - переписывает на цикловую
